Each target represents a system, or system component which has been analysed as part of the study. A target is a tuple of three sub-components:

  • The silicon device being analysed. This is usually "the microchip" in the system.
  • A specific core inside that device which has been analysed. This might be a CPU or hardware accelerator.
  • The circuit board and wider electronic system within which the device exists. This is recorded to capture differences in how different circuit setups for the same device might leak differently. For example, different setups in the power supply network. This also helps capture FPGA based platforms, where the same board and device might host many different cores.
Splitting the description of targets this way allows for finding differences between the same CPU, implemented on different devices. Or how differences in the wider system might cause the same device to leak differently.
